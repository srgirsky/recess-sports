# AGENTS.md — AI context for Recess Sports

The shared brief for EVERY coding agent on this project (Claude Code, Codex, ...). `CLAUDE.md` is a committed symlink to this file — same bytes, never edit them apart. Read this first. It's the fast on-ramp; deeper context is in `docs/OVERVIEW.md`, and human run/build/deploy steps are in `README.md`. **Keep all three current — see "Keeping docs current" at the bottom.**

## What this is (the one thing to understand)

A **free web baseball game for little kids (ages 4–8)**. You draft 9 of 30 neighborhood characters, then play a short game where you bat the top of each inning and pitch the bottom — one timing button for both. Balls in play become **live plays**: you steer a fielder with the pointer and charge throws on defense, and tap to send runners on offense (CLASSIC/KID MODE toggle on the title). The real product is **the 30 characters**: the game is a "voting machine" — every player draft pick is tallied, and pick rates reveal which kids should become toys/shows. So two jobs: (1) be genuinely fun, (2) log pick data. Design pillars: minimal reading, icon/voice-forward, short games.

## Stack

Phaser 3 · TypeScript · Vite · vitest. Static site, no backend, deployed free on **GitHub Pages** (auto-deploys on push to `main`; live at https://srgirsky.github.io/recess-sports/). All state (incl. pick tallies) is in `localStorage` for now.

## Architecture rules (follow these)

- **Pure game logic lives in `src/systems/` with NO Phaser imports.** Draft, at-bat resolution, innings, live plays, and pick logging are plain functions: state in → result out. They're unit-tested in `src/systems/logic.test.ts` + `liveplay.test.ts`. This is the most important rule — it keeps the tricky logic testable and lets the render loop stay dumb.
- **Scenes are the thin view layer.** They read input, call `systems/` reducers, and animate the result. A scene should never *decide* game outcomes — it plays back what a reducer returned. (E.g. baserunning animation on walks is driven by `ApplyResult.movements` from `inning.ts`, so it can't desync from the real base state.)
- **Balls in play run through the live-play sim** (`systems/liveplay.ts`): `resolveContact` (atbat.ts) turns a contact band into a trajectory, then GameScene's `update()` steps `stepLivePlay(state, inputs, dt)` every frame and positions everything the sim owns (ball, fielders, runners) **directly — never with tweens**. The sim-owned sprite layer lives in `scenes/ui/LivePlayView.ts` (sim-blind view: `buildDefense`/`beginPlay`/`render`/`reactTo`/`settlePlay`, plus the base rings and throw-charge meter); GameScene stays the controller — it steps the sim, drains events for stats/juice/booth calls (calling `view.reactTo(ev)` FIRST per event to preserve rng draw order), and owns `fieldAssignment`, `livePlay`, all input state, and the replay recorder. Outs emerge from the race. Kid mode: fly-outs + ball-beats-runner-to-the-bag only; caught fly returns runners free. Main mode (`params.manualBaserunning`): real rules — force-outs only when forced, tags for unforced runners (carrier within `RUN2.TAG_RADIUS` of an off-bag runner; rundowns emerge), tag-ups with doubling off, sac flies (a `CATCH_GATHER_MS` beat after catches keeps them winnable), per-runner send/hold via `LiveInputs.sendRunner/holdRunner` (tap a base ahead = send, behind = turn back). CPU policies for the non-human side live inside the sim. `finishLivePlay` → `applyLivePlay` (inning.ts) folds the result back into the half-inning. Field coordinates shared by sim + renderer live in `systems/geometry.ts`. The scene draws through the 3/4 camera in `art/projection.ts` (`project`/`unproject`/`depthScale`): the sim and ALL tests stay in flat logical space; only the render/input membrane projects (pointer events unproject on the way in). Never import projection from `systems/`. On top of it, GameScene runs **two views with a Backyard-style hard cut** (`config.PLATE_VIEW`): every pitch shows the **behind-home-plate rig** (`scenes/ui/BattingView.ts`) — a full-screen opaque venue-themed backdrop at depth `PLATE_VIEW.DEPTH` (50) with the batter big in the foreground in the rear-view `batRear` pose, the pitcher small and front-facing on a distant mound, the defending catcher (`catchRear`) cropped at the frame bottom, the other seven defenders small at their positions (`PLATE_VIEW.FIELDERS`, `ready` pose, fed from GameScene's `fieldAssignment`) so the close view shows the same defense as the wide field, and the ball flying AT the camera; on contact / runners moving / a steal race, `setView('wide')` hides the rig (white-flash cut) revealing the untouched 3/4 field. **The camera never pans or zooms** — `setView` is pure rig show/hide, and it must refresh the rig's actors on EVERY `'close'` call (batters change while the view stays close). Plate coords map to the frontal zone via `art/plateView.ts` (`plateToScreen`/`screenToPlate`, ~1.8× plate size); rig-era pointer input uses RAW screen coords (never `unproject`). HUD chrome renders on a dedicated UI camera via `pinUI()` (plus a corner mini-diamond inset: base state + the STEAL! chips); the pinned scoreboard strip (score panel / inning pill / labeled B-S-OUT pips with a change pulse + the HUD-anchored BALL!/STRIKE!/FOUL! umpire calls) lives in `scenes/ui/Scoreboard.ts`. Game mode (CLASSIC 'main' / KID MODE 'kid', persisted, main is the default) merges `config.LIVE` × `config.MODES[mode].live` via `systems/mode.ts`; `MODES[mode].features` flags gate the Backyard-style mechanics (all off in kid mode).
- **Team jerseys ride a texture-VARIANT resolver in `art/textureFactory.ts`**: `poseKey(id, pose)` silently appends a per-id suffix when `setTeamVariant(ids, teamSuffix(color, logo))` is armed, so the whole render layer (fielders, runners, rig, runCycle, reactions) wears team colors with zero per-call-site changes. LineupScene bakes the variant textures in the background (`queueTeamTextures` + `load.start()` — safe outside preload; `go()` waits for the loader); GameScene.init arms the resolver for both teams. **Street clothes are the same mechanism**: SchoolyardScene.create arms the whole roster with `STREET_SUFFIX` (`:sc`), so the entire draft (wall, crowd stream-out, inspect card, pennants) shows each kid's personal outfit (`VisualParams.outfit`: tee/stripeTee/hoodie/overalls/dress/jacket × `STREET_COLORS`); Boot bakes only the 4 draft poses (`STREET_POSES`), the G gallery lazily bakes the rest, and GameScene.init's `clearTeamVariant()` returns every scene after the draft — kid mode included — to jerseys (SeasonScene clears too, for the title→Week path). **Textures come in two sizes**: the base tier renders at 1.2× the viewBox (near display size — SVG-rasterizer downsampling beats GPU minification), and the rig-only hero tier (`:hi`, 2.4×, `HERO_POSES` = batRear/catchRear/swingMidRear/swingFollowRear/upset/nervous/cheer) serves the 230-288px behind-plate close-ups; `heroKey()` composes AFTER the team suffix and `BattingView` is its ONLY consumer. NEVER call `img.setTexture(plainId)` in game scenes — always `poseKey(id, 'stand')`, or the sprite silently drops its jersey. Logo emoji in the badge must be XML numeric entities (`badge()` in CharacterArt does this) — raw astral-plane chars mangle through the base64 data-URI path. |
- **Character art is generated once in `BootScene`** as modern toy-brand SVG (no image files): bold navy outlines with soft **gradient volume** inside them. **Lighting convention: one warm key light from the UPPER-LEFT, everywhere.** Shadows are cool (mixed toward navy via `darken`), highlights warm (`lighten`). `gradientDefs` in `CharacterArt.ts` builds per-document gradients from the kid's palette (radial `skinG`/`hairG` light-stop biased upper-left, linear `jerseyG`/`pantsG`); filled regions use them via `Ctx.gSkin`/`gJersey`/`gPants`. **Never put a gradient on a stroke** — `objectBoundingBox` gradients are undefined on zero-width bboxes (straight limbs); `capsule()` limbs keep their offset highlight-stroke instead. `hair()` takes pre-computed fill strings (deriving a shade from `'url(#...)'` NaNs — the art test catches it). The field agrees: the sun draws top-left, the mound is a lit dome, `shadeInt`/`lightenInt` in GameScene mirror the same mix. Keep any new art on this light or kids will look pasted on. `art/CharacterArt.ts` builds an SVG string from `VisualParams` (skin/hair/hairColor/uniform/accessory + `expression`/`bodyType`/`freckles`, plus the per-kid silhouette specs `body: BodySpec` — height/shoulderW/hipW/belly/neck/headW/headH, all range-CLAMPED in `buildBodySpec` so a content typo can't clip the viewBox — and `face: FaceSpec` — eyeGap/eyeSize/eyeStyle classic|button|sleepy, nose arc|dot|wedge, mouthW as a scaleX wrapper so the tested mouth paths never change, cheeks; `bodyType` is now just the preset the spec overrides. The outer height scale anchors at GROUND so feet stay planted; head geometry applies INSIDE `headGroup`/`headRearGroup` via `headXform`, with `wrapHeadBack` as the mandatory twin for separately-layered back hair) **and a `pose`** (`stand` front view, `run1`/`run2` side-view run cycle drawn facing right — flipX for left, `cheer` arms-up, `bat` side-view batting stance with the bat baked in (faces right — no flip at the plate), `windup` front-view leg-lift pitch, `ready` fielding crouch, `slide` low side-view with dust, the rear-view pair for the behind-plate rig — `batRear` batting stance from behind and `catchRear` catcher's crouch from behind, both built on `hairRear`/`headRearGroup`/`torsoRear` with NO `face()` call — the REACTION pair `upset` (strikeout slump) / `nervous` (bases-loaded fidget), front stands whose baked face OVERRIDES the kid's resting expression; swap them in one-shot via `anim.reactPose` (sprite-level) or `BattingView.reactBatter` (the rig batter turns to camera — wired at strikeouts and two-strike counts) — the ACTION set `throw` (side-view release, no baked ball), `catch` (front glove-up, mitt drawn OVER the head), and `dive` (prone side layout; the wheelchair kid leans-and-reaches), wired to the sim's throw/catch/dive events — and the SWING frames `swingMid` (contact: hips open, back heel up, barrel level through the zone with a white swoosh arc; the bat is FORESHORTENED — scaled about the grip — so a level barrel stays inside the viewBox) / `swingFollow` (weight transferred, barrel wrapped nearly horizontal behind the back — steeper angles hide the whole bat behind the head) plus rear twins `swingMidRear`/`swingFollowRear` for the rig, stepped load → contact → follow-through by `anim.poseSequence` (see the juice bullet). `VisualParams.stance` (`open`/`crouch`/`high`, optional) reshapes `poseBat`/`poseBatRear` per kid with NO extra textures — swing frames deliberately converge to ONE geometry (stance shapes only the load). **Hands are drawn, never circles**: `fist()` (palm block + thumb + knuckle hints; `gripPoints()` puts two of them ON the bat-handle axis so grips read wrapped) and `openHand()` (spread fingers for cheer/ready/catch/throw/slide/dive) — both follow the near-gradient/far-flat skin fill convention, and gloves always go through `mitt()`. **Hair layering rule: side poses draw `h.back` BEHIND the body and pass only `h.front` to the head — and any pose that MOVES the head (slide/dive) must carry `hBack` inside the head transform, or afros float loose / cover faces** (a regression test in art.test.ts enforces the order)); `art/textureFactory.ts` turns each into a Phaser texture (3× viewBox) keyed via `poseKey(id, pose)` — plain `id` for stand, `id:run1` etc. Animate runs by swapping textures with `anim.runCycle()`; put `anim.groundShadow()` under kids. Dev art gallery: press **G** on the Schoolyard, then **P** cycles poses, **A** animates the run (pick rates: **D**).
- **All "feel" tunables live in `src/config.ts`** (timing windows, pitch speed, innings, shake, runner speed, audio volume, the `VOICE` speech block: commentator profiles / derived kid-voice ranges / speech queue / chatter cadence, the `LIVE` live-play block, the `FLOW` pacing block, and the `MODES` kid/main table: live-sim multipliers + feature flags). Change feel there, not inside scenes. `FLOW` owns every between-moments beat (next pitch, next batter, half start, banner holds) — GameScene `delayedCall`s must use it, never literal ms; invariant: a banner's hold ≤ the FLOW beat that follows it, so calls finish before the next pitch.
- **Reusable juice** (shake/burst/floating text/confetti) is in `src/ui/effects.ts`; big-moment set pieces (homer show, juice-spend theatrics) in `src/scenes/ui/Spectacle.ts`; live-hit polish (streak trail, chalk landing ring, the `HIT_PAUSE_MS` contact-frame beat before the wide cut) is tuned in `config.FX`/`PLATE_VIEW`; **character animation** helpers (idle bob, squash-hop with `onDone`, pop-in, `enterFrom` staggered reveals, `pulse` attention loops, `groundShadow`, `runCycle` two-frame texture animation, `reactPose` one-shot reaction swaps, `poseSequence` one-shot multi-frame stepping with cancel) are in `src/ui/anim.ts`; **sound** (free, code-synthesized incl. the recess `bell()`) is in `src/systems/audio.ts`. Character motion combines pose-texture swaps (`runCycle`) with procedural tweens — e.g. the **swing** is a real frame sequence (stance → `swingMid` at `ANIM.SWING_CONTACT_FRAC` of `SWING_MS`, timed so the 90ms hit-pause flash catches the contact frame → `swingFollow` held `SWING_FOLLOW_MS`, longer + an over-rotation tween on a whiff) run by `GameScene.animateSwing(whiff)` / `BattingView.swingBatter(whiff)` through `poseSequence`, with a small body-whip tween on top; every path that re-poses or destroys the batter (`show`/`hide`/`reactBatter`/`showBatter`/`fadeOutBatter`) cancels the live sequence first, and nothing downstream ever waits on these timers. The pitcher gets a texture-swap **wind-up** the same way. The Schoolyard is title + draft in one continuous scene: PLAY rings the bell, doors burst open, kids stream to the brick wall (the stream-out is a real crowd sim — `systems/crowd.ts`, stepped from `SchoolyardScene.update()` with pairwise separation so kids never overlap; feel knobs in `config.CROWD`), tap-to-inspect stat cards, picked kids run to their team's pennant, both teams cheer. Other feel knobs live in `config.ANIM`.
- **Shared UI kit** is `src/ui/theme.ts`: the brand `FONT`, `OUTLINE`, and rounded-outlined-with-shadow helpers `panel()`, `ribbon()`, `pill()`, `heading()`. Use these (not raw `add.rectangle`) for any UI chrome — Phaser rectangles can't round corners. Buttons/cards (`ui/Button.ts`, `ui/CharacterCard.ts`) are built on it. The font is self-hosted Fredoka (`public/fonts/fredoka.woff2`, `@font-face` in `index.html`), awaited in `BootScene` before Title.

- **GameScene runs on a two-SEAT model.** `seats[0]` = away (the original "player" side), `seats[1]` = home (CPU); each `SeatState` carries team/score/lineupIdx/pitcher/plan/fatigue/juice/identity/stats plus `humanBats`/`humanPitches`/`recordsStats` flags. `battingSeat()`/`fieldingSeat()` resolve by half — BOTH flow families are now fully seat-parameterized: the human-batting family (`nextPlayerBatter`→`applyAndContinue`) and the human-pitching family (`nextCpuBatter`→`applyCpuResult`, incl. the relief helpers) read `battingSeat()`/`fieldingSeat()`, so either seat can hold the bat or the mound (pass-and-play flips batting; two-device play drives the mound remotely). Rules: credit runs to `battingSeat().score`, juice via `gainJuiceSeat(seat, …)` (the on-screen meter belongs to seat 0), stat events gated on `seat.recordsStats`, walk-off compares `seats[1]` vs `seats[0]` (home/away are seat POSITIONS, not roles). `enterHalf()` centralizes stale-timer/one-shot-input cancellation at every half boundary — add new cross-half timers there. Layout-only `half === 'top'` checks stay half-keyed. Regression: `scripts/goldlog.browser.js` + `scripts/goldlogs.json` (harness v2.1) — a seeded (mulberry32 + virtual `Date.now` locked to the pump grid) game drive whose state-transition log must stay byte-identical through behavior-identical refactors. The harness BYPASSES the visual draft with fixed teams and reseeds at game entry: `chooseBestPick` jitters every candidate with rng, so any title-screen cosmetic that consumes a draw would cascade into different drafted teams and a false-negative gate. **Cosmetics DO consume rng in one sneaky way: every `add.text(...)` creates a canvas texture whose key is a Phaser UUID drawn from `Math.random`** — so creating, removing, or reordering ANY Text object (incl. `floatingText`) relative to a sim rng call (`resolveContact` etc.) shifts the seeded stream and breaks the fingerprint; keep feedback-text calls on the same side of the resolve they're on today. It also never re-syncs the pump clock to `performance.now()` mid-run and samples on the 50ms step grid — real-async work (module parse, the jersey loader) must not shift the fingerprint.
- **Two-device play** (`matchType: 'net'` + `netRole: 'host' | 'guest'`; 🔗 FRIEND on the title): WebRTC via the free PeerJS cloud broker — still a zero-backend static deploy. `src/net/protocol.ts` is the PURE wire format (NetMsg union, HudSnap, validating encode/decode, Sequencer, emoji↔room-id — tested with a FakeTransport pair); `src/net/peer.ts` is the ONLY file importing peerjs (module-singleton `NetSession`: hostSession/joinSession/activeSession/dropSession). Topology: the room creator = **host = seats[0]** runs the one true sim with SOLO flow routing — the remote human enters at the two CPU decision seams (`launchPitch(Main)` parks for the guest's `pitchPlan`; `settleCpuPitch` routes to `settleNetSwing` for the guest's locally-resolved swing — an empty swing msg is a take; spends re-validated host-side via `canSpend`). The **guest never simulates**: `update()` early-returns into `netGuestUpdate` (streams `liveInput` @ `NET.FRAME_HZ`, renders `applyFrame(lerpFrames(prev, next, α))` through the same LivePlayView), and `handleNetMsg` mirrors host beats (half/settle/atBat/pitchLaunch/liveStart/liveFrame/liveEvents/gameOver) through existing methods; guest branches at the top of the four resolve hooks ship resolved bands/cursors instead of resolving — **every timing window resolves on the acting device**. Batting UI gates read `localHumanBats()` (net overrides seat flags); `deviceSeat()` is this device's fixed seat. Entry: LobbyScene (MAKE = 4 giant emoji; JOIN = 16-emoji grid; `NET.CODE_EMOJI`, hello/identity handshake — **no free text on the wire**, identities are color/logo indexes); the draft runs pick-by-pick over `draftPick{pickNo}` (each device `recordPick`s only its OWN taps) with a reliability layer — `send()` drops silently while the channel is down, so picks are ack'd (`draftAck`) and retransmitted (`PickCourier` in protocol.ts, `NET.DRAFT_RESEND_MS` on the scene clock) until acked, explicitly or implicitly by the friend's NEXT pick; duplicates are re-acked and ignored (`classifyDraftPick` — envelope seqs are fresh per retransmit, so the Sequencer can't dedupe them), only a FUTURE pickNo/pool miss aborts loud; the draft shows 🔍 during `reconnecting` and holds its exit until the final pick is acked; LineupScene edits both seats concurrently and exchanges `lineup` msgs. Disconnect: guest pauses under 🔍 immediately, host lets CPU policies finish the live play first; reconnect inside `NET.RECONNECT_MS` resyncs via a `resume` HudSnap; timeout/bye → no-blame GOOD GAME. Knobs in `config.NET`. v1 simplifications: remote batter steals are CPU-rolled, no remote steal-reaction bonus (wire fields reserved), guest manual relief off (CPU bullpen manages it), no net rematch.
- **Pass-and-play 2P** (`matchType: 'passplay'` on `GameInitData`; 👥 VS on the title): both seats get `humanBats` — BOTH halves run the seat-parameterized human-batting family against a CPU-run defense, and the human-pitching family is never entered. The batting player holds the device; `scenes/ui/HandoffSplash.ts` gates every half's FLOW timers behind a tap (`config.PASSPLAY.SPLASH_GUARD_MS`). Draft: both turns are human taps (both `recordPick` — real votes; the 'ai' turn is Player 2), no cpuScan. LineupScene runs twice (`pass: 1|2`; P2's identity is session-only + color-deduped). `deviceSeat()` = the batting seat in passplay (juice meter/chips follow the device); defensive CPU juice spends and the difficulty ramp are OFF in PvP; `recordGamePlayed` skipped; exhibitions only (no season). Scoreboard shows team logos/tints when identities exist (solo CLASSIC too); walk-off/Result name the winning team; the album credits both squads.

## Gotchas (things that will bite you)

- **Background tabs pause the game.** Browsers stop `requestAnimationFrame` when the tab isn't foreground, which freezes Phaser's clock — timers (AI pick, pitch) don't fire and input drops. Not a bug; playtest with the tab focused. For automated/headless visual checks, drive `window.__game` and force swings via `gameScene.resolvePlayerSwing(band, false)` (top half) or pitches via `gameScene.resolvePlayerPitch(band)` (bottom half — wait for `phase === 'aiming'`). **Pumping the clock correctly matters**: call `game.loop.step(t)` with monotonically increasing `t` (always > `game.loop.time` — screenshots can advance the real clock behind your back), NOT `game.step(t, delta)` (leaves `loop.delta` stale). And Phaser's tweens run on **wall-clock `Date.now()`**, not the game clock — to fast-forward tweens headlessly you must also rewind each active scene's `scene.tweens.startTime` by your step size per pump. Timers (`delayedCall`) follow the loop clock; tweens follow `Date.now()` — pump both or animations desync from logic.
- **Live plays are stepped from `GameScene.update()` on the game-loop clock — never tween anything the sim owns** (the live ball, fielders, runner tokens mid-play). Tweens are for chrome only (rings, meters, banners, exit fades). Headless: drive a live play with `gameScene.setLivePointer(x, y)`, `commandThrow(base, power)`, `commandRun()`, and inspect `getLivePlay()` while pumping the loop.
- **The Schoolyard stream-out is sim-owned, like live plays.** `SchoolyardScene.update()` steps `stepCrowd` only while `phase === 'cutscene'` (the scene lives through the whole draft — a stray step would fight idleBob/inspect tweens on the same containers). Never tween a kid's `root` mid-stream. The render-side run bob writes `kid.img.y` (the child inside the container) — zero it at the settle boundary before `idleBob` captures its baseline, or kids idle-bob around an offset. Kid draw order is `yardDepth(y)` (y-sorted, ≈11–34), not fixed constants.
- **Stop `runCycle` timers before destroying their sprite.** The cycle swaps textures on a repeating scene timer; destroying the image while the timer lives used to crash the whole game loop (`setTexture` on a dead sprite). `runCycle` now self-guards on `img.active`, but still `cycle.stop()` in teardown paths — and never re-start a cycle for a runner whose `done` is set (their exit fade owns the sprite). The cycle-owning live-play sprites (fielders + runner tokens) live in `scenes/ui/LivePlayView.ts` — its `buildDefense`/`settlePlay` teardowns are the canonical stop-before-destroy examples.
- **Phaser polygon points must be 0-based (no negative coords).** `add.polygon` computes its display origin from the AABB but does NOT normalize negative point coords, so negatives get shifted twice and the shape renders far from where you put it (this misplaced the infield dirt over the stands once). Author polygon point lists starting at 0,0. **`add.triangle` fills are similarly unreliable — draw filled/stroked shapes with Graphics** (`fillTriangle`/`strokeTriangle`), like `ui/theme.ts` does for rounded chrome. Also: shape color arrays need integer indexes — `bunt[(x / 56) % len]` silently yields `undefined` (invisible fill) when `x/56` is fractional; use `Math.floor`.
- **SVG textures need base64 data URIs.** Phaser's `load.svg` calls `atob` on data URIs — they must be base64 (`textureFactory.ts` does this), not URL-encoded, or it throws `InvalidCharacterError`.
- **`textures.getTextureKeys()` is polluted** by Phaser Text objects (GUID keys). Don't treat texture keys as character ids — use `ROSTER` from `data/characters.ts`.
- **The diamond is glued to the foul lines.** Bases lie ON the `FOUL_SLOPE` (1.2 dx/dy) line from home, and each venue's foul-pole x derives from its fence y via `foulPoleXAt` — so the drawn lines pass exactly through 1B/3B in every venue. Never reintroduce fixed pole x constants or move a base off the slope, and keep every `FIELD_POSITIONS` spot inside the fair cone for ALL venues (a geometry-sanity test in `liveplay.test.ts` enforces both). Size-coupled tunables (`RUNNER_SPEED`, `CPU_RUNNER_GREED_DIST`, `RUN2.CPU_PANIC_DIST`, `RUNNER_TWEEN_MS`) are scaled to the ~180px leg length — rescale them if the diamond changes size.
- **The fence arc must bulge AWAY from home (`fenceBulge ≥ 0`).** `fenceYAtX` being convex in x is what keeps `clampToField`'s convex-region argument valid — moveToward between two in-bounds points can never exit, and a caromed ball always reflects back into the field. A concave fence (negative bulge) breaks containment silently; a convexity property test in `venue.test.ts` enforces it.
- **All pose art must bottom out on the same ground line** (`GROUND = 248` in `CharacterArt.ts`): sprites use `setOrigin(0.5, 1)` and swap pose textures in place, so a pose whose lowest ink sits higher/lower makes the kid visibly pop up or sink when a run cycle starts. The per-kid height scale is anchored at `(100, GROUND)` for the same reason (an art test asserts it) — never re-anchor it at the viewBox center or short kids float.
- **Head geometry only scales through the headGroup-internal wrapper.** `headXform(c)` (neck lift + headW/headH about the head center) is emitted INSIDE `headGroup`/`headRearGroup`, so pose-level head transforms compose with it for free — but back hair layered BEHIND the body must wear the same transform via `wrapHeadBack(c, h.back)` at every concat site, or afros/drapes drift off scaled heads. Never scale hair or face outside that wrapper.
- **Never `clearTeamVariant()` on the Schoolyard without re-arming `:sc`.** The draft's street clothes ARE an armed texture variant; clearing it mid-draft silently flips every kid back to jerseys. Conversely any NEW scene reachable from the title that renders kids through `poseKey` inherits the street arm — clear it there if it's a jersey-era surface (SeasonScene does).
- **Character textures are sized near their display size on purpose** (base 1.2×, hero `:hi` 2.4× for the rig only). Don't "fix blur" by cranking the tiers back up — oversized textures get GPU-minified into mush (that WAS the old blur); if a new render site displays kids above ~200px, give it `heroKey` instead. Known limitation: the 960×640 framebuffer is CSS-upscaled on retina — that softening is renderer-level (DPR canvas + camera surgery), not a texture problem.
- **Phaser camera zoom ignores `setScrollFactor`** — HUD can't be pinned that way. GameScene runs a second UI camera: every screen-anchored element must go through `pinUI()` (it also routes container children — the input hit-tester checks them individually) or it will zoom with the field; an `addedtoscene` hook auto-hides everything else from the UI cam. Read world pointer coords via `cameras.main.getWorldPoint(p.x, p.y)`, NOT `pointer.worldX/worldY` — Phaser computes those against the topmost camera, which is the unzoomed UI cam.
- **GameScene has a scene-level `pointerdown` that swings/throws on ANY tap.** Corner HUD buttons (⏸ pause, 🔇 mute) must call `e.stopPropagation()` in their GameObject-level pointer handlers, or tapping the button mid-pitch also swings the bat.
- **Camera effects (`cam.pan`/`zoomTo`) need exact EaseMap keys** like `'Sine.easeInOut'` — tweens fuzzy-match the `'Sine.inOut'` shorthand, but camera effects don't resolve it, leave `this.ease` undefined, and crash the game loop on the next update. (GameScene no longer uses any — the pitch view is a rig cut, not a camera move — but SchoolyardScene's cutscene still does.)
- **Everything drawn during a pitch must sit ABOVE `PLATE_VIEW.DEPTH` (50)** — the rig backdrop is opaque, so a zone/ring/cursor/ball left at a legacy depth (13–20) silently vanishes under it. Rig-era visuals live at `DEPTH+2..+8`; `floatingText`/`burst` (60–80) already clear it. Related: `setView('close')` must refresh rig actors every call (it's called per-pitch while already close), and rig cursor input reads `lastScreenPointer` (raw screen coords) — running it through `toLogical`/`unproject` puts the cursor in the wrong place.
- **Audio needs a user gesture** to start (browser autoplay policy). It's unlocked on the Title PLAY click (`audio.unlock()`); all audio calls no-op before that or when muted, so they're always safe to call.
- **Net gotchas.** PeerJS ids must be ALPHANUMERIC — the emoji room code is UI-only; the wire id is `recess-` + 4 hex digits (`emojiToRoomId`). Every scene that subscribes to `NetSession.onMessage/onStatus` MUST unsubscribe on its `shutdown` event, or the dead scene's handlers fire on the next game's traffic. Heartbeat/staleness ride the PHASER clock (`session.tick(now)` from `update()` — the headless pump drives them; a throttled background tab stops ticking and trips the soft-disconnect on the healthy side, by design), while RECONNECT runs on wall-clock timers so it works under a frozen (paused) Phaser. Guests NEVER step the sim or schedule flow — `update()` early-returns and `startHalf` skips `begin()`; if a guest ever runs a `delayedCall` that resolves game state, the devices diverge. `features.replay` is forced off in net (frames already stream at 1×). Two tabs on ONE origin share localStorage — same-machine testing shares `recess_team`/picklog, real devices don't. Draft picks are the one ack'd+retransmitted message (see the two-device bullet) — the acks live in the `onMessage` CALLBACK, never the idle-gated drain (which stops running after the final pick), and any scene reachable right after the net draft must re-ack stray `draftPick`s (Lineup + Game do) or a lost final ack strands the sender. Bump `NET.PROTOCOL_VERSION` on ANY wire change — old deployed builds must be rejected at hello, not desync mid-game.
- **SpeechSynthesis is quirky.** `getVoices()` populates async (audio.ts re-caches on `voiceschanged` — the curated `rankVoices` ranking is cached with it and recomputed then; `unlock()` warms the cache on the PLAY gesture), Chrome drops an utterance spoken synchronously right after `cancel()` (the `flush` path defers one tick), and Chrome sometimes never fires `onend` — the speech queue has a duration watchdog so it can't wedge. Edge's "Online (Natural)" voices ignore the `pitch` param — speaker (and boy/girl) distinctness there comes from landing on different curated voices via the gendered sublists, not pitch; conversely Chrome desktop often ships no gender-marked voices at all, so there gender distinctness comes from the `GENDER_PITCH` bands on the mixed list. Speech volume is NOT governed by `AUDIO.masterVolume`; use `VOICE.VOLUME`. Never call `speechSynthesis` directly from scenes — go through `audio.say`/`audio.cancelSpeech` or the queue state desyncs.

## Keeping docs current (do this as part of each change)

Treat doc updates as part of the work, not an afterthought. After a change, update the doc that owns that fact:
- **New/changed commands or deploy steps** → `README.md`.
- **New scene/system/module, changed architecture, or a new gotcha** → this file's key-files map + gotchas, and `docs/OVERVIEW.md`.

One source of truth per fact + pointers between docs — don't duplicate, or they'll drift.
